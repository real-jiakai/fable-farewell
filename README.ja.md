# しばしの別れ、Fable 5 ⏳

Claude Fable 5 の Claude 各プランにおける統一的な標準搭載アクセスが終了する瞬間までの、非公式・多言語対応のカウントダウンページです。

## これは何か

このサイトは、Fable 5 の Claude 各プランにおける統一的な標準搭載アクセスが終了する瞬間——2026年7月19日、太平洋時間の午後11時59分59秒——までをカウントダウンしています。Fable 5 は**廃止や非推奨になるわけではありません**——2026年7月20日以降、Fable 5 は新しい、分割された体制に入ります。Anthropic が2026年7月18日に発表した @claudeai の告知によれば、すべての Max と Team Premium プランには上限の50%で引き続き搭載され（終了日は発表されていません）、Pro と Team Standard のユーザーは従量課金クレジットへ移行し、一度限りの100ドル分のクレジットを受け取ります。この間ずっと、Claude API では変わらず完全に利用可能で、これらの変更による影響は一切受けません。切り替えは当初2026年7月7日に予定されていましたが、この分割体制が発表される前にすでに二度延長されていました。まず2026年7月12日まで、続いて2026年7月19日まで延長されています。カウントダウンの対象時刻は `2026-07-20T06:59:59Z`（2026年7月19日、米国太平洋時間の午後11時59分59秒）です——これは Anthropic のサポート記事で公表された正式な時刻であり、推定値ではありません。

これは非公式のファンメイドページであり、Anthropic とは関係なく、公認されたものでもありません。

## 特徴

- サイト全体で単一の UTC 離脱時刻を保持し、クライアント側の `Intl.DateTimeFormat` で変換——タイムゾーン対応表を用意することなく、訪問者ごとに正しいローカル時間でカウントダウンを表示します。
- 初回描画から同期的に表示：カウントダウンはインラインスクリプトによって読み込み時にそのまま描画され、プレースホルダー数字のちらつきがありません。
- 英語・簡体字中国語・日本語をすべて完全に事前レンダリングした静的ルート（`/`、`/zh/`、`/ja/`）として提供し、`hreflang` タグで相互に関連付けています。
- Starlight スタイルの、アクセシブルなネイティブ `<select>` 言語切り替え。選択内容を記憶し、リダイレクトループを起こさずにブラウザの言語を自動検出します。
- 離脱の瞬間を7都市で示すワールドクロック。
- カウントダウン終了後の「離脱済み」状態への穏やかな切り替え。
- `prefers-reduced-motion` に対応した、CSS のみで実装された星空背景。
- フレームワーク・CDN・ウェブフォントを一切使用しない——ビルド成果物は完全に自己完結した静的サイトです。

## 技術スタック

- [Astro 7](https://astro.build)
- [Tailwind CSS 4](https://tailwindcss.com)（`@tailwindcss/vite` 経由）
- 素の インラインJavaScript（クライアントフレームワークや hydration なし）
- Node >= 22

## クイックスタート

```sh
npm install
npm run dev       # 開発サーバーを起動
npm run build     # 静的サイトを dist/ にビルド
npm run preview   # ビルド済みの dist/ をローカルでプレビュー
```

## プロジェクト構成

```
src/
├── i18n/
│   ├── config.js        # 唯一の情報源: DEPARTURE_ISO, DEFAULT_LOCALE, LOCALES, LOCALE_PATHS
│   └── ui.js             # en/zh/ja のテキスト辞書（本文、事実説明、ワールドクロックのラベルなど）
├── layouts/
│   └── Layout.astro      # <head>（meta、hreflang、favicon）+ ループしない言語自動リダイレクトスクリプト
├── components/
│   ├── Farewell.astro    # ページ本体：ヒーロー、カウントダウン、ワールドクロック、事実説明、フッター
│   └── LangSwitcher.astro# アクセシブルなネイティブ select 言語切り替え
└── pages/
    ├── index.astro       # en（デフォルト言語、プレフィックスなしルート）
    ├── zh/index.astro    # zh
    └── ja/index.astro    # ja
```

## 言語の追加方法

1. `src/i18n/ui.js` に新しい辞書エントリを追加する（既存のロケールの構造をコピーする）。
2. `src/i18n/config.js` の `LOCALES` と `LOCALE_PATHS` にその言語コードを追加する。
3. 既存のパターンに従い、`Layout` と `Farewell` に `locale` プロパティを渡すだけの薄いページを `src/pages/<コード>/index.astro` に追加する。

言語切り替えと `hreflang` タグは新しいロケールを自動的に認識します——他に言語を手動で列挙している箇所はありません。

## デプロイに関する注意

ビルド成果物（`dist/`）は完全に静的で、どこにでもホスティングできます。実際のドメインに公開する前に、`astro.config.mjs` に `site: 'https://…'` を設定し、canonical/hreflang を絶対URLにする必要があります——このサイトはまだ恒久的なドメインにデプロイされていないため、この作業は意図的に保留されています。
